# hPersist Smart Hands collector

This archive was generated by **hPersist**. Run it on a host with network
access to the iLO management interfaces of the target servers, then send back
`results.hpr`. The collector never talks to the internet — everything happens
locally on the host where you run it.

## 1 · Requirements

* Python **3.10 or newer** on Linux / macOS / Windows.
* Network reachability to every iLO over **HTTPS port 443**.
* The Redfish account on each iLO (read-only is enough; `Administrator` works).

Sanity check:

```sh
python3 --version          # 3.10+
curl -sk https://<one-ilo-ip>/redfish/v1/ | head
```

## 2 · Set up

```sh
# 1) create an isolated environment so we don't touch your system Python
python3 -m venv .venv
source .venv/bin/activate          # Windows:  .venv\Scripts\activate

# 2) install the two pinned dependencies
pip install -r requirements.txt
```

## 3 · Fill in the host list

Open `inventory.csv` and add one row per server. The header is fixed:

```
ip,hostname,login,password
10.0.10.42,db-fra-01,Administrator,YourPassword
10.0.10.43,,Administrator,YourPassword
```

* `hostname` is optional — leave it blank and we derive it from iLO.
* Lines starting with `#` are ignored, so feel free to leave the example rows
  in for reference.
* Save as **UTF-8** with **no BOM**.

> If the archive was generated *with* a pre-filled CSV, `inventory.csv`
> already contains your hosts. Edit only if something changed.

python collect.py

Useful flags:

| Flag              | Default | What it does                                     |
|-------------------|---------|--------------------------------------------------|
| `--concurrency`   | 8       | Parallel Redfish sessions. Lower for fragile iLOs.|
| `--timeout`       | 8       | Per-request timeout, seconds.                    |
| `--tls`           | warn-only | `strict` / `warn-only` / `off` — cert handling.|
| `--inventory PATH`| inventory.csv | Use a different CSV.                       |
| `--output PATH`   | results.hpr | Write the envelope somewhere else.           |

## 5 · Send back the result

Hand `results.hpr` to the hPersist operator (encrypted email, S3 presigned
URL, USB drop — your call). On their side:

> **Add inventory → Smart Hands → Process results → drop the file.**

The operator will see the integrity verification chain (signed per-host
records), and any modifications to this collector script will be surfaced as
an informational warning. The result is still usable; the warning is there so
the operator knows what they're looking at.
